# Contributing to PANOPTIA

Thank you for your interest in contributing to PANOPTIA.

## How to contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/my-feature`
3. Make your changes and add tests where applicable
4. Run tests: `npm test`
5. Ensure TypeScript compiles: `npm run build`
6. Commit using a clear message: `git commit -m "feat: add X"`
7. Open a pull request against `main`

## Code style

- TypeScript strict mode is enabled
- Follow existing file/module structure in `src/`
- No `any` types without justification

## Reporting issues

Open a GitHub Issue with:
- Describe the bug or feature request
- Reproduction steps (if a bug)
- Expected vs actual behaviour

## Security

Do NOT open public issues for security vulnerabilities. Send details to the maintainers privately via GitHub private advisory.
